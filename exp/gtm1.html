GTM 自訂事件

2017-07-13


TL;DR

在 GTM 裡，如果要追蹤某個元素的點擊狀況大家都會；但如果要追蹤網頁中某個區塊的點擊狀況 ( 類似熱點 )，難不成要一個元素下一個 tag，下到整個版面都是嗎？那還要用 GTM 幹嘛。

這時候就換 自訂事件 + dataLayer 登場了。


push 資料進 dataLayer

首先我們要知道什麼是 dataLayer： 一個 GTM 內部的全域變數。
更進一步探索它的架構，可以在網頁上使用開發者工具的 console 欄，輸入 dataLayer

你會發現 GTM 測試欄位左邊有幾個事件，dataLayer 裡面就有幾個 object。
再把 object 點開 (這裡以 window loaded 為例)，會有 event 和 GTM 給的事件 id，而這個事件( gtm_load )其實就是「觸發條件」裡的「視窗已載入」。

因此，我們可以自己寫一段觸發條件，然後 push 進 dataLayer ，就可以在 GTM 裡面操作了： See the Pen dataLayer。


這裡我們想偵測 nav 裡面所有連結點擊的狀況，並回傳裡面的文字。
"event": "menu_click" 就是我們自訂的觸發條件名稱，而 "menu_text": $text 就是「變數」裡面的 dataLayer (資料層變數)。

程式端完成之後，就可以回到 GTM 介面開始設定了↓

GTM 設定

如果想要抓的東西在更深的子層，可以選版本 2 往子層找。
example: gtm.element.children.0.alt( 抓取元素裡圖片的 alt 屬性 )，這個通常運用在單一元素的點擊事件。


以前單純用 GA 的時候，其實程式端只要在 js 加寫一行當前網址 $url，然後 push 的地方改成
ga('sent','event','電子報 menu click',$text, $url) 就把上面的事情都做完了。現在如果觸發條件或變數不是 GTM 裡頭的預設值時，那就使用 dataLayer 吧，所有事件都可以以此類推。
